Rendering Statistics Window

The Game View has a Stats button top right. When this Stats button is pressed, an overlay window is displayed with realtime rendering statistics. This is very useful for Optimizing iPhone Graphics Performance.

Rendering Statistics Window

Statistics window contains the following information:

Emulated Draw CallsPredicts how many draw calls will be necessary to render this frame on the iPhone.
Batched (Draw Calls)Predicts how many draw calls could be batched together on the iPhone. Batching means that engine will be able to combine rendering of multiple objects into one draw-call which leads to lower CPU overhead. To ensure good batching you should share as many materials between different objects as possible.
Editor Draw CallsNumber of draw calls used to render this frame in the Editor
Tris and VertsNumber of triangles and vertices drawn. Note that degenerate triangles are also included in these statistics. Unlike desktop GPUs degenerate triangles have performance impact on iPhone therefore they have to be counted.
Used TexturesCount and memory size of textures used when drawing this frame.
Render TexturesCount and memory size of Render Textures that are created. Render Textures are not supported in Unity iPhone yet.
ScreenSize and memory taken by the screen itself.
VRAM usageApproximate bounds of current video memory (VRAM) usage. Also shows how much video memory your graphics card has.
VBO totalNumber of unique meshes (vertex buffers). Each different model will cause new VBO to be created. In some cases scaled objects will cause additional VBOs to be created. In case of a static batching however number of objects can share same VBO.
Visible Skinned MeshesHow many skinned meshes are rendered.
AnimationsHow many animations are playing.
