Components are attached to game objects. Attaching a Renderer component to a game object is what makes it render on the screen, attaching a Camera turns it into a camera object. All scripts that can be attached to an object are also components.

The most common components are accessible as simple member variables:

ComponentAccessible as
Transformtransform
Rigidbodyrigidbody
Rendererrenderer
Cameracamera (only on camera objects)
Lightlight (only on light objects)
Animationanimation
Collidercollider
... etc.

For the full list of predefined member variables see the documentation for the Component, Behaviour and MonoBehaviour classes. If the game object does not have a component of the type you wish to fetch, the above variables will be set to null.

In order to access another component that is not directly accessible through a member variable (like another script attached to the same game object), you can use the GetComponent function like this:

var otherScript : OtherScript;
otherScript = GetComponent(OtherScript);

Note that the first line in the above example explicitly defines the type of the variable otherScript using the ... : typename construct. This is because the type of the variable would otherwise be set to the return type of GetComponent.
