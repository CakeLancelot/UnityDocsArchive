Interface into the Input system.

Members

Class Variables

Class Functions

Use this class to read the axes set up in the Input Manager.

By default every project has the following axes defined: "Horizontal" and "Vertical" are mapped to joystick, A, W, S, D and the arrow keys. "Mouse X" and "Mouse Y" are mapped to the mouse movement. "Fire1", "Fire2" "Fire3" are mapped to Ctrl, Alt, Cmd keys and three mouse or joystick buttons. New input axes can be added in the Input Manager.

If you are using input for any kind of movement behaviour use Input.GetAxis. It gives you smoothed and configurable input that can be mapped to keyboard, joystick or mouse.

Use Input.GetButton for action like events only. Don't use it for movement, Input.GetAxis will make the script code smaller and simpler.

Class Variables

NameDescription
mousePosition

The current mouse position in pixel coordinates.

anyKey

Is any key currently held down?

anyKeyDown

Returns true the first frame the user hits any key.

inputString

Returns the keyboard input entered this frame. Only ASCII characters are contained in the inputString.

Class Functions

NameDescription
GetAxis

Returns the value of the virtual axis identified by axisName.

GetButton

Returns true if the virtual button identified by buttonName is held down.

GetButtonDown

Returns true the first frame the user hits the virtual button identified by buttonName.

GetKey

Returns whether the key identified by name is held down. Think auto fire.

GetKeyDown

Returns true the first frame the user hits the key identified by name.

GetMouseButton

Returns whether the the given mouse button is held down.

GetMouseButtonDown

Returns true the first frame the user hits the given mouse button.

ResetInputAxes

Resets all input. After ResetInputAxes all axes return to 0 and all buttons return to 0 for one frame.
