Using .NET API 2.1 compatibility level

Now Unity iPhone supports two .NET API compatibility levels: .NET API 1.1 and .NET API 2.1. You can select appropriate level in PlayerSettings.

.NET API 1.1

Pros:
  1. Smallest application build size.
  2. Fastest application startup time.
Cons:
  1. Troubles when porting game scripts from desktop Unity.
  2. Standard .NET API 1.1 libraries offer less features.
  3. No generics and other new language features support.

Unity iPhone API compatibility settings default to .NET API 1.1.

.NET API 2.1

Unity iPhone supports mixed profile of .NET APIs 2.0 and 2.1. It is closest to the Mono "monotouch" profile, so many limitation that are applicable to the "monotouch" profile are also applicable for Unity iPhone implementation of this .NET profile. More information on "monotouch" profile limitations can be found here.

Pros:
  1. Better code compatibility with desktop Unity and third party libraries
  2. More features in standard API set.
  3. Generics support.
Cons:
  1. Application build size is bigger.
  2. Slightly worse application startup time.

Note: Unity iPhone doesn't support packages in the scripts. If you have third party library with the source then the best practice is to compile this library outside Unity iPhone Editor and drop library .dll file into Assets folder.
