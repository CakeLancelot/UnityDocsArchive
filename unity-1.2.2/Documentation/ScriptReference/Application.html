Application.runInBackground

bool runInBackground

Should the player be running when the application is in the background? Default is false.

Application.runInBackground = true;

Application.dataPath

string dataPath

Contains the path to the game data folder. The value depends on which platform you are running on:

Unity Editor:<path tp project folder>/Assets
Mac player:<path to player app bundle>/Contents
Win player:<path to dir containing the player exe>\Data
Dasboard widget:<path to the dashboard widget bundle>
Web player:empty string - The web player does not allow access to local files.

Application.LoadLevel

void LoadLevel (int index)

Loads the level by its index. You can see the indices of all levels using the the Build Settings File->Build Settings... menu. Before you can load a level you have to add it to the list of levels used in the game. Use the menu item File->Build Settings... and add the levels you need to the level list there.

// Loads the level with index 0
Application.LoadLevel (0);
When loading a new level all game objects that have been loaded before are destroyed. If you want to let an object survive when loading a new level, use Object.DontDestroyOnLoad.

Application.LoadLevel

void LoadLevel (string name)

Loads the level by its name. Before you can load a level you have to add it to the list of levels used in the game. Use the menu item File->Build Settings... and add the levels you need to the level list there.

// Load the level named "HighScore".
Application.LoadLevel ("HighScore");
When loading a new level all game objects that have been loaded before are destroyed. If you want to let an object survive when loading a new level, use Object.DontDestroyOnLoad.

Application.CaptureScreenshot

void CaptureScreenshot (string filename)

Captures a screenshot at path filename as a png file. If the file exists already, it will be overridden.

function OnMouseDown() {
    Application.CaptureScreenshot("Screenshot.png");
}
