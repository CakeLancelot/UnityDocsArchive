Introduction

Unity Manual > User Guide > Graphics > Introduction

At the core of the graphics engine lies a system that is as simple as it is powerful: Filters and Pipelines. All graphics in unity is processed through a collection of filters (called a pipeline). A filter is a small component that takes some graphics data in, modifies it and spits it out to the next filter in the pipeline - small pieces of code that can modify gemoetry.

At the end of a pipeline is a Renderer - this takes the resulting graphics from the pipeline and renders it on screen.

Filters fall into 3 categories:

Producers
These filters produce geometry. They typically load a mesh from disk and insert it into a pipeline.
Modifiers
These filters modift whatever is passed to them. This could be a scale filter that changes the size of an object, or deformations.
Readers
These filters take whatever is passed to them. Examples include ((comp-CollisionFilter|Collision Filter), that takes whatever graphic is in the pipeline and makes object in the world collide with it.

Together, the different components of filter system provides a huge configurability in the graphics system, allowing for just about any effect imaginable.
