Class Variables

Debug.isDebugBuild

bool isDebugBuild

In the Build Settings ... dialog there is a check box called "strip debug symbols". If it is checked isDebugBuild will be off. In the editor isDebugBuild always returns true. It is recommended to remove all calls to Debug.Log when deploying a game, this way you can easily deploy beta builds with debug prints and final builds without.

// Log some debug information only if this is a debug build
if (Debug.isDebugBuild) {
    Debug.Log ("Something bad happened!");
}

Class Methods

Debug.DrawLine

void DrawLine (Vector3 start, Vector3 end, Color color = Color.white)

Draws a line from the point start to end with color. The line will be drawn in the scene view of the editor. If gizmo drawing is enabled in the game view, the line will also be drawn there.

// Draws a red line from the the world-space origin to the point (1, 0, 0)
function Update () {
    Debug.DrawLine (Vector3.zero, new Vector3 (1, 0, 0), Color.red);
}

Debug.DrawRay

void DrawRay (Vector3 start, Vector3 dir, Color color = Color.white)

Draws a line from start to start + dir with color.

// Draws a 10 meter long green line from the position
// forward along the transform's z-axis.
function Update () {
    var forward = transform.TransformDirection(Vector3.forward) * 10;
    Debug.DrawRay (transform.position, forward * 10, Color.green);
}

Debug.Log

void Log (object message, Object context)

Logs message to the Unity Console. When you select the error in the console a connection to the context Object will be drawn. This is very useful if you want know on which object an error occurrs.

Debug.Log ("Hello", gameObject);
