Variables

RenderTexture.width

int width

The width of the render texture in pixels. Note that unlike Texture's width property, this is both read and write - setting a value changes size.

RenderTexture.height

int height

The height of the render texture in pixels. Note that unlike Texture's height property, this is both read and write - setting a value changes size.

RenderTexture.depth

int depth

The depth of the render texture in bits (0, 16, 24, 32 are supported)

RenderTexture.isPowerOfTwo

bool isPowerOfTwo

Is the size of the render texture constrained to power of two sizes? When using image post processing effects, you should always disable this (use non-power of two textures). When using the render texture on normal shaders you should always enable this (use power of two textures).

Class Variables

RenderTexture.active

RenderTexture active

The active render texture. All rendering goes into the active RenderTexture. If the active RenderTexture is null everything is rendered in the main OpenGL window. When a RenderTexture becomes active its rendering context is automatically created if it hasn't been created already.

RenderTexture.enabled

bool enabled

Is render texture enabled? When RenderTexture is disabled, no rendering contexts are created and rendering will always go into the main OpenGL window.

Class Methods

RenderTexture.GetTemporary

RenderTexture GetTemporary (int width, int height, int depth = 0)

Allocate a temporary render buffer. This function is optimized for when you need a quick buffer to do some temporary calculations. Release it using RenderTexture.ReleaseTemporary as soon as you're done with it, so another call can share the buffer.

RenderTexture.ReleaseTemporary

void ReleaseTemporary (RenderTexture temp)

Release a temporary buffer allocated with RenderTexture.GetTemporary Later calls to RenderTexture.GetTemporary will reuse the RenderTexture. When no one has requested the the temporary RenderTexture for a few frames it will get destroyed.
