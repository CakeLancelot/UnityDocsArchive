Materials and Shaders

Unity Manual > User Guide > Graphics > Materials and Shaders

Unity has an extensive shader system, allowing you to tweak the look of all in-game graphics. It works like this:

A shader basically defines a formula for how the in-game shading should look. Each shader also has a number of properties (typically textures). Materials are created from shaders, and define which textures and colors are used by the shaders. The materials are attached to game objects, defining the look in the game.

As this is rather complex, A graph is in order:

On the left side of the graph there is the Carbody shader. 2 Materials are created from this: Blue car material and Red car material. Each of these have 2 textures assigned: The car texture that defines the main texture of the car, and a color FX texture. These are used by the shader to make the car finish look like 2-tone paint (this can be seen on the front of the red car: it is yellow where it faces the camera and then fades towards purple as the angle increases). The car materials are assigned to the 2 cars. The car wheels, lights and windows don't have the color change effect, and must hence use another material. At the bottom of the graph there is a Simple Metal Shader. From this, the Wheel Material is made. Note that even though the Car texture is reused here, the end result is quite different from the car body, as another shader is used.

Note: Quite a few textures, shaders & materials were left out from this diagram in order to avoid clutter.

To be more specific, a shader defines:

  • The method to render an object. This includes using different methods depending on the graphics card of the end user.
  • Any vertex and fragment programs used to render.
  • Some texture properties that are left up to materials to fill in.
  • Color and number settings that materials need to provide

A Material defines:

  • Which textures to use for rendering.
  • Which colors to use for rendering.

Shaders are meant to be written by graphic programmers. They are written using the ShaderLab scripting language, which is quite simple. However, getting a shader to work well on all graphic cards is an involved job, and requires a fairly comprehensive knowledge of how graphic cards work.

A number of standard shaders come with Unity in the Standard Asset Library.

Making new Materials

To make a new Material, either:

  • Right-click on a Shader in the library window and select 'Add Material'
  • Select an existing material and choose duplicate from the edit menu.
  • Drag a texture onto a mesh. This automatically creates the material in the 'Materials' folder next to the texture.

Setting Material Properties

When you select an object in the scene, you can pick the Material in the component pop-up, or selecting a material from the library. After doing this, the Material inspector pops up. This is split into 2 areas: at the top, all properties of the material is listed. The properties currently available are numbers, colors, sliders and textures. As you change these, you can see the changes to the scene updated in real-time.

To assign a texture, drag one from the library over one of the texture buttons. To edit the mapping options for the texture, click the texture button. The bottom part of the inspector will show the mapping options for the selected texture:

Property:Function:
Scale Scales the texture along the different axes
Offset Slides the texture around

Note: If a shader uses vertex programs, the scale and offset properties have no effect

Changing The Shader of a Material

Sometimes, you wish to change the shader being used by a material. In order to do so, simply drag a shader on the material in the library window. Unity will carry across any properties that the shaders share. All shaders that ship with unity have been constructed with this in mind, so most of the time you can just replace a shader and see the effect.
