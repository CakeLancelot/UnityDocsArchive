Most advanced game code does not only manipulate a single object. The Unity scripting interface has various ways to find and access other game objects:

1. Located through the object hierarchy.

You can find child and parent objects to an existing object through the Transform component of a game object:

// Moves all transform children 10 units upwards! for (var child : Transform in transform) {
child.position += Vector3.up * 10.0;
}

See the documentation for the Transform class for further information.

2. Passed as parameters.

Some event messages contain detailed information on the event. For instance, trigger events pass the Collider component of the colliding object to the handler function.

function OnTriggerEnter( other : Collider ) {
if (other.transform.root != transform.root) {
// the colliding objects are not part of the same hierarchy... // insert scollision handling code, such as damage calculations, here }
else {
// the colliding objects are part of the same hierarchy - ignore }
}

Note that by suffixing the other variable in the above example, you can access any component inside the colliding object.

3. Through global variables and functions.

The main camera object is accessible through Camera.main. You can search for game objects with certain tags using GameObject.FindWithTag and GameObject.FindGameObjectsWithTag. You can also use the Object.FindObjectOfType to locate objects other than game objects. Use GameObject.Find to find a game object by name.

function Start ()
{
// By name var go = GameObject.Find("SomeGuy");
go.transform.position.y = 5;

// By tag var player = GameObject.FindWithTag("Player");
player.transform.position.y = 5;

}
