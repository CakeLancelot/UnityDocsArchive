Getting Started with iPhone Development

Note: If you have not yet set up your Apple Developer account, go here first and follow the instructions. You cannot use Unity iPhone to its full potential without an Apple Developer account.

Developing for the iPhone

Building games for a device like the iPhone requires a different approach than targeting the home PC market. The device's hardware is standardized, and not as fast or powerful as a computer with a dedicated video card. Because of this, you'll have to consider new things while developing your games. There are some feature differences in the iPhone version of Unity as well.

Setting up your Apple Developer account

Before you can run Unity iPhone games on the actual device, you'll need to have your Apple Developer account approved and set up. This includes setting up your team, adding your devices, and finalizing your provisioning profiles. All this setup is performed through Apple's developer website. Since this is a complex process, we've provided a basic outline of what tasks must be completed before you can run code on your iPhone or iPod Touch. However, the best thing to do is follow the instructions step-by-step from Apple's iPhone Developer portal.

Access iPhone unique functionality

Unity iPhone provides a number of new scripting APIs to access multi-touch screen, accelerometer, device geographical location and much more. You can find out more about new scripting classes on iPhone Scripting page.

Exposing native C, C++ or Objective-C code to scripts

Unity iPhone allows you to call custom native functions written in C, C++ or Objective-C directly from C# scripts. To find out how to bind native functions, visit the Plugins page.

Occlusion Culling

Unity iPhone includes support for Occlusion Culling, which is perfect for squeezing high performance out of complex scenes with many objects. To learn how Occlusion Culling works, visit the Occlusion Culling feature page.

Splash Screen customization

If you want to customize the splash screen that appears when launching your game on the device, you should visit Splash Screen customization page.

Troubleshooting

If your game crashes on a device, there might be several reasons why. Learn how to troubleshoot on iPhone.

How Unity iPhone differs from desktop Unity

.NET 1.1 (No Generics)

Unity iPhone currently supports only .NET 1.1. This means there is no access to generics or other .NET 2.0 advanced features. When you launch existing projects that use Generics you will receive compiler errors. You need to convert your code to not use Generics or .NET 2.0 features.

Strongly typed JavaScript

Unity iPhone has dynamic typing always turned off in JavaScript. This is the same as adding #pragma strict to your scripts, but instead it is done by default. This greatly improves performance which is especially important on the iPhone. When you launch an existing Unity Project and you are using dynamic typing, you will receive compiler errors. You can easily fix them by statically typing the variables that give you errors or taking advantage of type inference.

Terrain Engine

The Terrain engine is not supported on the iPhone. The Terrain engine is built to work with modern PC GPUs and does not sensibly scale down to the iPhone. Using custom low poly meshes and occlusion culling is the best way to create terrains on the iPhone.

MP3/AAC instead of Ogg Vorbis audio compression

Runtime Ogg Vorbis decompression is very expensive for the iPhone's processor and is disabled as such. When building for the iPhone, you will receive a build error if any of your project audio files are set to use Ogg Vorbis compression. You can switch these files to use uncompressed audio in the import settings. For background music you can use .mp3 or .m4a compressed audio files. Consult the AudioClip documentation for more information on usage of compressed audio for the iPhone.

PVRTC instead of DXT texture compression

Unity iPhone does not support DXT textures. Bring up the import settings and select PVRTC texture compression which is natively supported on the iPhone and compresses the textures both in memory and distribution size.

Movie playback

Movie Textures are not supported on iPhone, instead a full-screen streaming playback is provided via scripting functions. To learn about supported file formats and scripting API, consult the Movie page.
