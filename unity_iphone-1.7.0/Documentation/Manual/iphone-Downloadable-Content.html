Preparing your application for "In App Purchases"

This chapter does not aim to cover how to integrate your game with Apple "StoreKit" API. It is assumed that you already have integration with "StoreKit" either via native code plugins.

Apple "StoreKit" documentation defines four kinds of Products that could be sold via "In App Purchase" process:

  • Content
  • Functionality
  • Services
  • Subscriptions

This chapter covers the first case only and focuses mainly on downloadable content concept.AssetBundles are ideal candidates for use as downloadable content and there will be covered:

  • How to export asset bundles for use on iPhone
  • How download and cache them on iPhone

Exporting your assets for use on iPhone

Having separate project for downloadable content could be good idea allowing better separated between content that comes with your main application and content that is downloaded later.

  1. When creating new dedicated project for downloadable content don't forget to include iPhone Standard Assets

    This will include Editor helper scripts for asset export.
  2. Design your objects that needs to be downloadable as prefabs
  3. Select prefab that needs to be exported and mouse right click

    If iPhone Standard Assets were included then Build AssetBundle From Selection - Track dependencies and Build AssetBundle From Selection - No dependency tracking context menu items should be seen.
  4. Select the first one if you want to include everything what this asset uses and second one if you want your assets to share some elements which will be exported separately.
  5. Save dialog will be shown, enter desired asset bundle file name. .assetbundle extension will be added automatically. Unity iPhone runtime accepts only asset bundles built with the same version of editor as final application is built. Read BuildPipeline.BuildAssetBundle for details.

Downloading your assets on iPhone

  1. Asset bundles can be downloaded and loaded via WWW class and instantiate main asset. Code sample:
    	var download : WWW;

	var url = "http://somehost/somepath/someassetbundle.assetbundle";

	download = new WWW (url);

	yield download;

	assetBundle = download.assetBundle;

	if (assetBundle != null)
	{
		// Alternatively you can also load an asset by name (assetBundle.Load("my asset name"))
		var go : Object = assetBundle.mainAsset;

		if (go != null)
			instanced = Instantiate(go);
		else
			Debug.Log("Couldnt load resource");	
	}
	else
	{
		Debug.Log("Couldnt load resource");	
	}
  2. You can save required files to a Documents folder next to your game's Data folder.
            public static string GetiPhoneDocumentsPath () 
        { 
                // Your game has read+write access to /var/mobile/Applications/XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX/Documents 
                // Application.dataPath returns              
                // /var/mobile/Applications/XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX/myappname.app/Data 
                // Strip "/Data" from path 
                string path = Application.dataPath.Substring (0, Application.dataPath.Length - 5); 
                // Strip application name 
                path = path.Substring(0, path.LastIndexOf('/'));  
                return path + "/Documents"; 
        }
  3. Cache downloaded asset bundle using .NET file API and for reuse it in the future by loading it via WWW class and file:///pathtoyourapplication/Documents/savedassetbundle.assetbundle. Sample code for caching:
    	// Code designed for caching on iPhone, cachedAssetBundle path must be different when running in Editor
       // See code snippet above for getting the path to your Documents folder
	private var cachedAssetBundle : "path to your Documents folder" + "/savedassetbundle.assetbundle"; 
	var cache = new System.IO.FileStream(cachedAssetBundle, System.IO.FileMode.Create);
	cache.Write(download.bytes, 0, download.bytes.Length);
	cache.Close();
	Debug.Log("Cache saved: " + cachedAssetBundle);
